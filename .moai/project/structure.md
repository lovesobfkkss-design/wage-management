# 프로젝트 구조

## 디렉토리 트리

```
급여관리/
├── index.html              # 메인 진입점 (HTML 구조)
├── css/
│   └── style.css           # 전체 스타일링 (848줄)
├── js/
│   ├── app.js              # 메인 앱 로직 (2,131줄)
│   ├── data.js             # 데이터 관리 & Firebase 연동 (538줄)
│   ├── utils.js            # 유틸리티 함수 (346줄)
│   └── firebase-config.js  # Firebase 설정 (17줄)
├── .moai/                  # MoAI-ADK 설정
├── .claude/                # Claude AI 설정
└── .github/                # GitHub 워크플로우
```

## 주요 파일 설명

### index.html
- 앱의 진입점
- 로그인 화면, 메인 앱, 모달 구조 정의
- Firebase SDK 및 JS 파일 로드 순서 관리

### js/app.js (메인 로직)
- **UI 렌더링**: 탭별 화면 구성
- **이벤트 핸들링**: 버튼 클릭, 폼 제출
- **탭 관리**: 대시보드, 직원관리, 비율제강사, 근무기록, 급여정산, 문자생성, 설정
- **역할별 화면**: 관리자/직원 분기 처리

### js/data.js (데이터 레이어)
- **Firebase 연동**: 실시간 데이터 동기화
- **CRUD 함수**: 직원, 근무기록, 비율제강사 관리
- **데이터 호환성**: 버전 업그레이드 시 마이그레이션
- **내보내기/가져오기**: JSON, CSV 처리

### js/utils.js (유틸리티)
- **상수**: 최저시급, 기본값
- **계산 함수**: 급여, 공제, 근무시간
- **포맷팅**: 날짜, 금액, 시간
- **CSV 처리**: 변환, 다운로드

### js/firebase-config.js
- Firebase 프로젝트 설정
- 앱 초기화

### css/style.css
- CSS 변수로 테마 관리
- 반응형 레이아웃
- 컴포넌트별 스타일

## 데이터 흐름

1. **초기 로드**: localStorage에서 캐시 데이터 로드
2. **Firebase 동기화**: 서버 데이터와 실시간 동기화
3. **사용자 액션**: UI 이벤트 → data.js 함수 호출
4. **저장**: localStorage + Firebase 동시 저장
5. **화면 갱신**: 데이터 변경 시 자동 리렌더링
